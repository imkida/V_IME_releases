macOS Beta · 语音输入与文字整理

在 Mac 上，把想法直接说进输入框。

VIME 是一个桌面语音输入工具。按下右 Command 说话，VIME 会完成语音转写、文本整理、自动翻译或选中文本编辑，并把结果写入当前输入位置。

内部测试说明： VIME macOS 目前由个人开发，供内部测试用户提前试用。测试过程中如果遇到问题、不稳定或体验不符合预期，请多包涵；有问题可以在飞书联系 @kk.wang。

开始使用

首次启动只需要完成权限、服务和快捷键三件事。未配置个人 API Key 时，也可以先使用 VIME 默认云端服务试用。

安装并启动

下载 DMG，把 V_IME.app 拖到 Applications，从应用程序文件夹启动。建议不要从下载目录或 DMG 里直接长期运行。

完成系统权限

允许麦克风用于录音；允许辅助功能用于把转写结果写回当前输入框。VIME 会在设置页显示权限状态。

确认服务配置

默认云端服务可直接使用。你也可以在“服务配置”中保存自己的 API Key，个人 Key 会优先于默认服务。

用右 Command 输入

在任意输入框内按住或双击右 Command 开始听写。完成后，VIME 会把整理后的结果写入光标位置。

快捷键

默认快捷键围绕右 Command 设计，尽量保持单手操作。左 Command 仍保留给系统和常见应用快捷键。

输入模式

右⌘按住

快速说一句，松开后转写并写入。

右⌘双击

进入连续听写；再次单击或双击右 Command 结束。

右⌘,

启动中英互译。中文自动转英文，英文自动转中文。

右⌘.

启动文本编辑。先选中文本，再口述修改要求。

输出风格

状态栏面板可以切换输出风格，快捷键也可以快速切换。

  • 智能整理：整理成清晰正文或要点。
  • 保留语气：轻微修正错字、标点和断句，保留原本说法。
  • 逐字稿：尽量保留语音原文。

1.1.0 beta 更新内容

这个 hotfix build 在 1.1.0 beta 基础上修复写回退化为字母 a 候选的问题，并继续收口默认云端服务、快捷键冲突、写回稳定性和学习体验。

默认云端服务

未配置个人 API Key 时，可以使用 VIME 默认云端服务；保存个人 Key 后会优先使用个人服务。

右 Command 快捷键

优化按住说话、连续听写、自动翻译、文本编辑和输出风格切换，降低误触发。

写回 hotfix

修复部分输入法 / WebView 环境下 Unicode 写回退化成字母 a 候选的问题，并增强写回失败检测。

写回稳定性

改进聊天输入框、Web / Electron 应用和长文本写入，降低 E401、焦点漂移和剪贴板竞态。

常用词和学习

学习体验保持可感知、可撤销、可删除；普通用户不会看到工程调试信息。

输出策略对齐

继续对齐 Android Clean / Tidy 基线，优先保证意图保真，再做段落整理和压缩。

发布链路

本 hotfix 包使用 Developer ID 签名，并通过更新 manifest 向测试用户分发；本轮不走 Apple notarization。

常见问题

如果遇到写入失败、权限未生效或网络不可用，可以先按下面方式排查。

提示辅助功能未授权

在系统设置中允许 VIME 使用辅助功能。授权后通常会自动生效；如果长时间仍不可用，再完全退出并重新打开 VIME。

没有写入目标输入框

先确认光标在目标输入框内。若 VIME 无法安全写回，会保留结果并提示复制，避免说过的内容丢失。

默认服务不可用

请检查网络。也可以在“服务配置”中保存自己的 API Key，个人服务会优先使用。

右 Command 和外接键盘冲突

部分外接键盘、远程桌面或改键工具可能拦截右 Command。可以在设置中配置备用快捷键。
